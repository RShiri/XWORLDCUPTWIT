Method · xG model

How my expected-goals model works

Expected goals (xG) is the probability an average shot from a given situation ends in a goal. My model reads a set of engineered features off each shot -

Distance to goalShooting angleHeader vs. foot Assist / pass typeOpen play vs. set piece

- and runs them through two learners in parallel: an L2-regularised logistic regression and a gradient-boosted tree ensemble (LightGBM) with monotonic constraints, so closer, straighter chances can never be scored below worse ones. Their outputs are blended in log-odds space at the weight that minimises out-of-fold error.

I anchor the shape to bookmakers' and providers' public xG by distillation - a student model learns their values from the same features, so it still works on shots that were never covered - while real goals keep the overall level honest. A final isotonic calibration, always fit out-of-fold to avoid optimism, plus a per-league level shift, forces the model's summed xG to match the goals that were actually scored.

Deliberately no resampling or class-weighting for the ~10% goal rate: reweighting inflates the probabilities, and an honest probability is the whole point. The trained model exports to a small JSON artifact scored by a dependency-free Python runtime - the same one that powers the dots and numbers above, and my Barça xG Lab.
